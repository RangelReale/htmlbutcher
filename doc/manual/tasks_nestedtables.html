HTMButcher Manual

Nested tables
One of the big differences of HTMLButcher to other slicing applications, is the possibility to create nested-tables. Normally image slicing applications only generate a single big HTML table containing the full image layout. Although this works for a lot of sites, for some complex designs, or ones that will by used in dynamic sites, this limits the amount of changes you can do on the layout later, without breaking the layout.

In HTMLButcher, you can create unlimited tables-inside-tables, or in HTMLButcher nomenclature, areas inside areas. Each inside area will translate to another HTML table inside a parent table's cell.
To do this, right-click on an area, and set its "Area Kind" to "Mask".



On this not-too-useful example layout, "Area 7" is a cell of the main layout's table. Inside this cell there will be a nested table, shown here by the "Area 1" area "Area 3" just below the "Area 7" title. (The area 2 is the small area between area 1 and area 2, the title is not shown because the cell width is too short).
The "Area 1" cell inside "Area 7" was divided in 2 by a vertical line, and the right area was given one more inner table.

This will translate to the following HTML code: 
  <TR>
    <!-- AREA 7 -->
    <TD  WIDTH="980" HEIGHT="155" VALIGN="TOP" STYLE="background-repeat: no-repeat; "> 


          <TABLE WIDTH="980" CELLSPACING="0" CELLPADDING="0">
            <TR>
              <!-- AREA 7 => AREA 1 -->
              <TD  WIDTH="236" HEIGHT="155" VALIGN="TOP" STYLE="background-repeat: no-repeat; "> 
          

                    <TABLE WIDTH="236" CELLSPACING="0" CELLPADDING="0">
                      <TR>
                        <!-- AREA 7 => AREA 1 => AREA 1 -->
                        <TD  WIDTH="66" HEIGHT="155" VALIGN="TOP" STYLE="background-repeat: no-repeat; "> 
                          <IMG SRC="images/home_area7_area1_area1.jpg" WIDTH="66" HEIGHT="155" BORDER="0"></TD>
                        <!-- AREA 7 => AREA 1 => AREA 2 -->
                        <TD  WIDTH="170" HEIGHT="155" VALIGN="TOP" STYLE="background-repeat: no-repeat; "> 
                    

                              <TABLE WIDTH="170" CELLSPACING="0" CELLPADDING="0">
                                <TR>
                                  <!-- AREA 7 => AREA 1 => AREA 2 => AREA 1 -->
                                  <TD  WIDTH="170" HEIGHT="98" VALIGN="TOP" STYLE="background-repeat: no-repeat; "> 
                                    <IMG SRC="images/home_area7_area1_area2_area1.jpg" WIDTH="170" HEIGHT="98" BORDER="0"></TD>
                                </TR>
                                <TR>
                                  <!-- AREA 7 => AREA 1 => AREA 2 => AREA 2 -->
                                  <TD  WIDTH="170" HEIGHT="57" VALIGN="TOP" STYLE="background-repeat: no-repeat; "> 
                                    <IMG SRC="images/home_area7_area1_area2_area2.jpg" WIDTH="170" HEIGHT="57" BORDER="0"></TD>
                                </TR>
                                <!-- alignment row -->
                                <TR>    <TD  WIDTH="170" HEIGHT="1">    </TD>  </TR>
                              </TABLE>

                    
                        </TD>
                      </TR>
                      <!-- alignment row -->
                      <TR>    <TD  WIDTH="66" HEIGHT="1">    </TD>    <TD  WIDTH="170" HEIGHT="1">    </TD>  </TR>
                    </TABLE>

          
              </TD>

For areas of kind "Mask", you can still generate a background image for them, selecting "Background image" from the area's right click menu. You can then set the top-level area as kind "None", this way the area will be transparent and will show the background image of the table below it.

Using a combination of nested masks and background-repeated images, you can create a streachable area that adjusts for any resolution that the browser may run in. See the tutorial on streatchable areas to learn how to do this.
